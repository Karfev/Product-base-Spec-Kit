## 1. Назначение услуги

Предоставление Потребителям облачной платформы виртуального центра обработки данных (ВЦОД) и вычислительных ресурсов по модели Infrastructure-as-a-Service (IaaS).

## 2. Описание технологии предоставления

### 2.1 Архитектура технического решения

> Детальная архитектура продукта: [`products/private-cloud/architecture/overview.md`](../../products/private-cloud/architecture/overview.md)

ИТ-инфраструктура Частного облака РТК ЦТ (ЧО РТК-ЦТ, Система) предназначена для организации облачной вычислительной инфраструктуры и состоит из оборудования, прикладного и общесистемного ПО. Система включает следующие компоненты:

- **Вычислительная инфраструктура** — физические серверы для внутренних нужд (кластер управления) и серверы для предоставления вычислительных ресурсов Потребителю (кластеры вычислений)
- **Инфраструктура хранения данных** — разделяемые сети и системы хранения данных с логическим выделением ресурсов (зонирование SAN, выделение разделов СХД)
- **Подсистема виртуализации VMware** — компоненты управления средой виртуализации, гипервизоры кластеров управления и вычислений
- **Телекоммуникационная инфраструктура** — логически изолированные сегменты разделяемой сетевой фабрики
- **Подсистема обеспечения информационной безопасности** — виртуальные МСЭ разделяемой подсистемы межсетевого экранирования; сертификат ФСТЭК России (К1/ГИС, УЗ2/ПДн)
- **Подсистема мониторинга** — круглосуточный мониторинг компонентов инфраструктуры
- **Подсистема инфраструктурных сервисов** — вспомогательные сервисы
- **Подсистема Единого портала** — единый графический интерфейс управления
- **Подсистема резервного копирования** — защита данных

**Смежные системы:**

- **Инженерная инфраструктура ЦОД** (электроснабжение, климат, противопожарная защита, физический контроль доступа) — обеспечивается и обслуживается ООО «РТК-ЦОД»
- **Система Service-Desk** — приём, регистрация и обработка обращений

**Внешние каналы связи** обеспечивают связь Системы с внешними сетями передачи данных и с глобальной публичной сетью Интернет. Предоставляются и обслуживаются ПАО «Ростелеком».

**Подход к сегментированию ресурсов для потребителей:**

- Вычислительная инфраструктура — физическое выделение серверов; разделяемые кластеры вычислений между несколькими Потребителями
- Инфраструктура хранения данных — логическое выделение ресурсов (зонирование + отдельные разделы)
- Подсистема виртуализации VMware — отдельные компоненты управления, физическое выделение гипервизоров
- Телекоммуникационная инфраструктура — логическое выделение изолированных VLAN-сегментов
- Подсистема ИБ — логическое выделение виртуальных МСЭ

### 2.2 Ресурсно-сервисная модель

> Полная РСМ с Mermaid-диаграммами: [`rsm.md`](./rsm.md)

Ресурсно-сервисная модель описывает девять подсистем, обеспечивающих предоставление услуги ВЦОД-VMWR-VS:

| Подсистема | Описание |
|---|---|
| Телекоммуникационная инфраструктура | Сетевая фабрика, VLAN, МСЭ, каналы Интернет |
| Инфраструктура хранения данных | СХД, SAN-зонирование, диски трёх типов |
| Подсистема ИБ | Виртуальные МСЭ, подсистема защиты VMware |
| Платформа виртуализации VMware (кластер управления) | vCenter, компоненты управления |
| Платформа виртуализации VMware (кластер вычислений) | Гипервизоры, ресурсные пулы |
| Подсистема мониторинга | Система мониторинга метрик и событий |
| Подсистема инфраструктурных сервисов | DNS, NTP, Active Directory и др. |
| Подсистема резервного копирования | Средства резервного копирования данных |
| Подсистема Единого портала + терминальные серверы | Консоль управления ВЦОД, VPN-доступ |

## 4. Регламенты взаимодействия

Обработка обращений пользователей Единой службой технической поддержки производится в соответствии с внутренними регламентами:

- **Управление инцидентами** — внутренний регламент ООО «РТК-ЦТ»
- **Управление запросами на обслуживание** — внутренний регламент ООО «РТК-ЦТ»
- **Взаимодействие с Заказчиком** — Регламент обработки обращений пользователей (разрабатывается после заключения SLA)

SLT для инцидентов — в [`ops/incident-catalog.yml`](./ops/incident-catalog.yml).
SLT для запросов — в [`ops/request-catalog.yml`](./ops/request-catalog.yml).
