# NFR-профиль домена: Характеристики качества ПО по ГОСТ Р ИСО/МЭК 25020

**Домен:** is-ontology
**Версия:** 1.0.0
**Последнее обновление:** 2026-03-03
**Источник:** ГОСТ Р ИСО/МЭК 25020-2023 «Системная и программная инженерия. Требования и оценка качества систем и программной продукции (SQuaRE). Основные принципы измерения качества». Онтология АИС «Налог-3», ГНИВЦ / ФНС, 2025, Раздел 5.

---

## 1. Описание и назначение

Данный NFR-профиль определяет стандартный набор характеристик и подхарактеристик качества ПО для использования в requirements.yml в части NFR-требований. Каждая характеристика соответствует классификатору ГОСТ Р ИСО/МЭК 25020-2023.

**Как использовать:**
1. При описании NFR-требования в requirements.yml укажите `type: nfr`
2. В поле `metrics` добавьте измеримые показатели из данного профиля
3. Укажите `iso_characteristic` для однозначной привязки к стандарту

---

## 2. Рекомендуемые характеристики качества

Ниже представлены характеристики и подхарактеристики, рекомендованные к использованию согласно онтологии АИС.

### 2.1 Функциональная пригодность

**iso_key:** `functional_suitability`

Степень соответствия функций ПО заявленным и подразумеваемым нуждам.

| Подхарактеристика | iso_key | Описание | Пример метрики |
|-------------------|---------|----------|----------------|
| Функциональная полнота | `functional_completeness` | Степень, в которой набор функций охватывает все задачи и цели пользователя | % охваченных user stories |
| Функциональная корректность | `functional_correctness` | Степень корректности результатов или эффектов | % успешных тест-кейсов |
| Функциональная уместность | `functional_appropriateness` | Степень, в которой функции облегчают выполнение конкретных задач | Экспертная оценка |

**Рекомендовано к применению:** функциональная полнота, функциональная корректность.

---

### 2.2 Уровень производительности (Эффективность)

**iso_key:** `performance_efficiency`

Производительность по отношению к используемым ресурсам при определённых условиях.

| Подхарактеристика | iso_key | Описание | Пример метрики |
|-------------------|---------|----------|----------------|
| Временные характеристики | `time_behaviour` | Степень соответствия времени отклика и пропускной способности требованиям | P95 latency, RPS, throughput |
| Использование ресурсов | `resource_utilization` | Степень соответствия объёма и типов ресурсов требованиям | CPU %, RAM MB, disk I/O |
| Ёмкость | `capacity` | Степень соответствия максимальных пределов параметру требований | Макс. кол-во одновременных сессий |

**Рекомендовано к применению:** временные характеристики.

---

### 2.3 Совместимость

**iso_key:** `compatibility`

Степень способности ПО обмениваться информацией с другими системами.

| Подхарактеристика | iso_key | Описание | Пример метрики |
|-------------------|---------|----------|----------------|
| Функциональная совместимость | `interoperability` | Степень способности системы обмениваться и использовать информацию с другими | % успешных интеграционных тестов |
| Соседство | `co-existence` | Степень способности ПО выполнять задачи, делясь ресурсами с другими системами | % случаев деградации при совместной работе |

**Рекомендовано к применению:** функциональная совместимость.

---

### 2.4 Удобство использования

**iso_key:** `usability`

Степень, в которой продукт может использоваться для достижения целей с эффективностью, результативностью и удовлетворённостью.

| Подхарактеристика | iso_key | Описание | Пример метрики |
|-------------------|---------|----------|----------------|
| Защищённость от ошибки пользователя | `user_error_protection` | Степень защиты пользователей от ошибочного ввода | % ошибочных операций, приводящих к неверным данным |
| Распознаваемость пригодности | `appropriateness_recognizability` | Степень, в которой пользователи могут осознать, подходит ли продукт для их нужд | NPS, CSAT |
| Доступность и обучаемость | `learnability` | Степень простоты использования для достижения задач | Время до первого успешного завершения задачи |

**Рекомендовано к применению:** защищённость от ошибки пользователя.

---

### 2.5 Надёжность

**iso_key:** `reliability`

Степень выполнения системой заявленных функций при заданных условиях в течение заданного времени.

| Подхарактеристика | iso_key | Описание | Пример метрики |
|-------------------|---------|----------|----------------|
| Отказоустойчивость | `fault_tolerance` | Степень работоспособности при наличии аппаратных или программных ошибок | Uptime %, % успешных запросов при сбое ноды |
| Восстанавливаемость | `recoverability` | Степень восстановления данных и состояния после отказа | RTO (Recovery Time Objective), RPO |
| Зрелость | `maturity` | Степень соответствия потребностей в надёжности при обычной эксплуатации | MTBF (Mean Time Between Failures) |
| Доступность | `availability` | Степень работоспособности и доступности системы при необходимости использования | Uptime %, SLA % |

**Рекомендовано к применению:** отказоустойчивость, восстанавливаемость.

---

### 2.6 Защита (Безопасность)

**iso_key:** `security`

Степень защиты информации и данных от несанкционированного доступа.

| Подхарактеристика | iso_key | Описание | Пример метрики |
|-------------------|---------|----------|----------------|
| Конфиденциальность | `confidentiality` | Степень доступности данных только авторизованным лицам и системам | % случаев несанкционированного доступа к данным |
| Целостность | `integrity` | Степень предотвращения несанкционированного изменения компьютерных программ или данных | % несанкционированных изменений за период |
| Неотказуемость | `non-repudiation` | Степень, в которой действия или события могут быть доказаны произошедшими | % операций с аудит-трейлом |
| Подотчётность | `accountability` | Степень прослеживаемости действий сущности | % действий с полным аудит-логом |
| Аутентичность | `authenticity` | Степень доказуемости идентичности субъекта или ресурса | % успешных аутентификаций с MFA |

**Рекомендовано к применению:** конфиденциальность.

---

### 2.7 Сопровождаемость

**iso_key:** `maintainability`

Степень эффективности и рациональности, с которой продукт может быть модифицирован.

| Подхарактеристика | iso_key | Описание | Пример метрики |
|-------------------|---------|----------|----------------|
| Модульность | `modularity` | Степень, в которой система состоит из дискретных компонентов | % компонентов с независимым развёртыванием |
| Повторная используемость | `reusability` | Степень использования актива в нескольких системах | % кода, используемого более чем в одном контексте |
| Анализируемость | `analysability` | Степень простоты оценки влияния изменений | Среднее время диагностики инцидента (MTTD) |
| Модифицируемость | `modifiability` | Степень простоты изменения без добавления дефектов | Среднее время реализации изменения |
| Тестируемость | `testability` | Степень простоты определения критериев тестирования и выполнения тестов | Покрытие тестами (code coverage %) |

---

### 2.8 Переносимость

**iso_key:** `portability`

Степень переноса системы из одной среды в другую.

| Подхарактеристика | iso_key | Описание | Пример метрики |
|-------------------|---------|----------|----------------|
| Адаптируемость | `adaptability` | Степень адаптации к различным средам | Количество ручных шагов при смене среды |
| Устанавливаемость | `installability` | Степень простоты установки/удаления в заданной среде | Время развёртывания в новой среде |
| Взаимозаменяемость | `replaceability` | Способность заменить другой продукт для тех же целей | % функциональных тестов, проходящих на обоих продуктах |

## 3. Формат для requirements.yml

Для NFR-требований с привязкой к ГОСТ 25020 используйте следующий шаблон:

```yaml
- id: REQ-NFR-001
  title: Время отклика API при пиковой нагрузке
  type: nfr
  priority: P1
  status: proposed
  description: >
    P95 времени ответа на API-запросы не должно превышать 200 мс при
    нагрузке 1000 запросов в секунду.
  metrics:
    - name: api_response_p95_ms
      target: 200
      measurement: APM (Prometheus histogram, p95)
      iso_characteristic: performance_efficiency
      iso_sub_characteristic: time_behaviour
  trace:
    slo:
      - ops/slo.yaml#api-response-latency
    tests:
      - tests/perf/api-latency.jmx
```

```yaml
- id: REQ-NFR-002
  title: Доступность сервиса
  type: nfr
  priority: P0
  status: proposed
  description: >
    Сервис должен быть доступен не менее 99.9% времени (допустимое
    плановое недоступное время — не более 8.7 часов в год).
  metrics:
    - name: service_availability_percent
      target: 99.9
      measurement: Uptime monitoring (Blackbox exporter, 1-minute probes)
      iso_characteristic: reliability
      iso_sub_characteristic: availability
  trace:
    slo:
      - ops/slo.yaml#service-availability
```

```yaml
- id: REQ-NFR-003
  title: Конфиденциальность персональных данных
  type: nfr
  priority: P0
  status: proposed
  description: >
    Персональные данные пользователей не должны передаваться
    неавторизованным третьим лицам или логироваться в plaintext.
  acceptance_criteria:
    - Секреты API-ключей не появляются в логах
    - Персональные данные передаются только в зашифрованном виде (TLS 1.2+)
    - Аудит-лог содержит все операции с ПД
  metrics:
    - name: unauthorized_pii_access_count
      target: 0
      measurement: SIEM (событийный мониторинг)
      iso_characteristic: security
      iso_sub_characteristic: confidentiality
  trace:
    tests:
      - tests/security/pii-leak-scan.sh
```

## 4. Минимальный набор NFR для профиля Enterprise IS

Следующие характеристики **MUST** быть определены в requirements.yml при profile=enterprise:

| Характеристика | iso_characteristic | Обязательность |
|----------------|-------------------|----------------|
| Временные характеристики | performance_efficiency / time_behaviour | MUST |
| Доступность | reliability / availability | MUST |
| Отказоустойчивость | reliability / fault_tolerance | MUST |
| Восстанавливаемость | reliability / recoverability | MUST |
| Конфиденциальность | security / confidentiality | MUST |
| Функциональная совместимость | compatibility / interoperability | SHOULD |
| Тестируемость | maintainability / testability | SHOULD |

## 5. Матрица рисков и характеристик

| Зона риска | Приоритетные характеристики |
|------------|----------------------------|
| Публичный API / внешние интеграции | performance_efficiency, compatibility, security |
| Обработка персональных данных | security (confidentiality, integrity, non-repudiation) |
| Высоконагруженные сервисы (>1000 RPS) | performance_efficiency, reliability (availability, fault_tolerance) |
| Критические бизнес-процессы | reliability (recoverability, maturity), functional_suitability |
| Компонент с внешними зависимостями | compatibility (interoperability), reliability (fault_tolerance) |
| Долгосрочное сопровождение | maintainability (modularity, testability, analysability) |
