# Каталог взаимосвязей: Таксономия архитектурных связей

**Домен:** is-ontology
**Версия:** 1.0.0
**Последнее обновление:** 2026-03-03
**Источник:** Онтология АИС «Налог-3», ГНИВЦ / ФНС, 2025. ArchiMate 3.2.

> **Примечание:** В отличие от event-catalog доменных событий (domain events), этот файл описывает **типы архитектурных взаимосвязей** между элементами модели АИС. В терминологии SpecKit они аналогичны «событиям» — триггерам и связям, управляющим поведением системы.

---

## 1. Общая классификация взаимосвязей

Все взаимосвязи делятся на три класса:

```
Взаимосвязи
├── Структурные (моделируют статическое строение)
│   ├── Реализации
│   ├── Назначения
│   ├── Агрегации
│   └── Композиции
├── Зависимостей (моделируют использование и поддержку)
│   ├── Ассоциативная
│   ├── Доступа
│   └── Обслуживания
└── Динамические (моделируют поведенческие зависимости)
    ├── Запускающая
    └── Потоковая
```

**Общие ограничения:**
- Между двумя взаимосвязями не допускается использовать ещё одну взаимосвязь
- Взаимосвязь, соединяющая элемент со второй взаимосвязью, может быть только взаимосвязью композиции или ассоциативной взаимосвязью
- Каждая взаимосвязь должна исходить «от» какого-либо элемента и доходить «до» какого-либо элемента

---

## 2. Структурные взаимосвязи

Моделируют статическое строение или композицию понятий.

### 2.1 Взаимосвязь реализации

**Смысл:** Один элемент реализуется в другом элементе.

**Нотация:** Стрелка с пустым треугольником на конце

**Типичные применения:**
- Объект данных реализует объект деятельности
- Система/подсистема реализует интерфейс деятельности
- Компонент приложения реализует процесс деятельности
- Узел реализует компонент приложения

**Чтение:** «[Элемент A] реализуется посредством [Элемент B]» или «[Элемент A] реализуется через [Элемент B]»

**Пример в тексте:** «Программный интерфейс для платных абонентов реализуется посредством Веб-интерфейса»

---

### 2.2 Взаимосвязь назначения

**Смысл:** Один элемент назначен другому элементу (назначен на него, размещён на нём).

**Нотация:** Стрелка с заполненным ромбом у источника

**Типичные применения:**
- Артефакт назначен (размещён) на узел
- Компонент приложения назначен на узел

**Правило:** Может быть выражена и путём вложенности элементов модели (элемент вложен внутрь контейнера)

---

### 2.3 Взаимосвязь агрегации

**Смысл:** Элемент объединяет один или несколько других элементов. Агрегирующий и агрегированный элементы могут существовать независимо.

**Нотация:** Линия с пустым ромбом у агрегирующего элемента

**Типичные применения:**
- Объект деятельности 1 объединяет Объект деятельности 2 и Объект деятельности 3
- Группировка объединяет несколько подсистем

**Правило:** Элемент, соприкасающийся с ромбом, является агрегатом для элемента, на который указывает окончание взаимосвязи.

---

### 2.4 Взаимосвязь композиции

**Смысл:** Элемент состоит из одного или нескольких элементов. При удалении родительского элемента дочерние элементы (как правило) тоже удаляются.

**Нотация:** Линия с заполненным ромбом у составного элемента

**Отличие от агрегации:** Выражает зависимость от существования другого элемента.

**Типичные применения:**
- Узел состоит из системного ПО
- Процесс деятельности 1 состоит из Процессов деятельности 2 и 3

**Правило:** Элемент, соприкасающийся с чёрным ромбом, состоит из элементов, на которые указывает окончание взаимосвязи.

---

## 3. Взаимосвязи зависимостей

Описывают, как одни элементы поддерживают другие или используются ими.

### 3.1 Ассоциативная взаимосвязь

**Смысл:** Общая зависимость, не охватываемая никакими другими взаимосвязями. Используется, когда характер связи неизвестен или незначим.

**Нотация:** Простая линия (без специальных маркеров)

**Правило:** Допустима между двумя элементами или между взаимосвязью и элементом. По умолчанию ассоциативная взаимосвязь является ненаправленной.

---

### 3.2 Взаимосвязь доступа

**Смысл:** Поведенческий или активный структурный элемент обращается к пассивному структурному элементу (читает, записывает, обновляет).

**Нотация:** Пунктирная стрелка с открытым концом, с указанием типа доступа

**Типы доступа:**
- `read` (R) — чтение
- `write` (W) — запись
- `read-write` (RW) — чтение и запись
- `create` (C) — создание
- `delete` (D) — удаление

**Типичные применения:**
- Компонент приложения читает объект данных
- Сервис записывает в объект данных

---

### 3.3 Взаимосвязь обслуживания

**Смысл:** Один элемент обслуживает (предоставляет услуги) другому элементу. Связь отображается от элемента, который **служит**, в сторону элемента, **которому служат**.

**Нотация:** Стрелка с закрытым треугольником

**Правило чтения:** «[Элемент A] служит [Элемент B]» → стрелка от A к B.

**Типичные применения:**
- Интерфейс приложения служит участнику деятельности
- Интеграционная платформа сервисов служит ФОИВ (предоставляет возможность запрашивать сведения)
- Внешняя система служит разрабатываемой системе

---

## 4. Динамические взаимосвязи

Используются для моделирования поведенческих зависимостей между элементами.

### 4.1 Запускающая взаимосвязь (Triggering)

**Смысл:** Один элемент поведения инициирует (запускает) другой элемент поведения.

**Нотация:** Стрелка с заполненным наконечником

**Типичные применения:**
- Событие деятельности запускает процесс деятельности
- Завершение одного процесса запускает следующий процесс
- Входящий запрос запускает функцию обработки

---

### 4.2 Потоковая взаимосвязь (Flow)

**Смысл:** Передача данных или управления от одного элемента к другому в рамках одного или более процессов.

**Нотация:** Пунктирная стрелка с открытым наконечником

**Типичные применения:**
- Передача объекта деятельности между двумя процессами
- Потоки данных на схемах потоков данных (DFD)

## 5. Сводная таблица всех взаимосвязей

| № | Класс | Взаимосвязь | Нотация | Направление | Типичный смысл |
|---|-------|-------------|---------|-------------|----------------|
| 1 | Структурная | Реализации | Стрелка с пустым треугольником | A → реализуется → B | B реализует поведение A |
| 2 | Структурная | Назначения | Стрелка с заполненным ромбом | A ← назначен ← B | B размещён на A |
| 3 | Структурная | Агрегации | Линия с пустым ромбом | A ◇— B | A объединяет B |
| 4 | Структурная | Композиции | Линия с заполненным ромбом | A ◆— B | A состоит из B |
| 5 | Зависимости | Ассоциативная | Простая линия | A — B | Общая нетипизированная связь |
| 6 | Зависимости | Доступа | Пунктирная стрелка с типом | A --[R/W/RW]--> B | A обращается к B |
| 7 | Зависимости | Обслуживания | Стрелка с треугольником | A → служит → B | A предоставляет услуги B |
| 8 | Динамическая | Запускающая | Стрелка заполненная | A →→ B | A запускает B |
| 9 | Динамическая | Потоковая | Пунктирная стрелка открытая | A --> B | Данные/управление от A к B |

## 6. Правила применения взаимосвязей по слоям

| Взаимосвязь | Деятельности → Приложений | Приложений → Технологический | Внутри одного слоя |
|-------------|--------------------------|-----------------------------|--------------------|
| Реализации | ✓ (компонент реализует процесс) | ✓ (узел реализует компонент) | ✓ |
| Назначения | — | ✓ (артефакт на узле) | ✓ |
| Агрегации | — | — | ✓ |
| Композиции | — | ✓ (узел содержит системное ПО) | ✓ |
| Ассоциативная | ✓ | ✓ | ✓ |
| Доступа | ✓ (функция читает объект) | ✓ | ✓ |
| Обслуживания | ✓ (интерфейс служит участнику) | ✓ | ✓ |
| Запускающая | — | — | ✓ |
| Потоковая | ✓ | — | ✓ |

## 7. Рекомендации по выбору взаимосвязи

При проектировании схем используйте следующий алгоритм выбора:

1. **Если один элемент физически воплощает другой** → Взаимосвязь реализации
2. **Если элемент размещён/назначен на другой элемент** → Взаимосвязь назначения
3. **Если элемент содержит части, которые не могут существовать отдельно** → Взаимосвязь композиции
4. **Если элемент объединяет части, которые могут существовать отдельно** → Взаимосвязь агрегации
5. **Если сервис/интерфейс предоставляет возможности пользователю или другой системе** → Взаимосвязь обслуживания
6. **Если элемент читает/записывает данные** → Взаимосвязь доступа
7. **Если одно событие/действие запускает другое** → Запускающая взаимосвязь
8. **Если данные передаются от одного элемента к другому** → Потоковая взаимосвязь
9. **Если ни одна из выше не подходит** → Ассоциативная взаимосвязь
