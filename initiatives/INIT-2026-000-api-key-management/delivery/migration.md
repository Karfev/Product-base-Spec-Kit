# Migration Plan: INIT-2026-000-api-key-management

## Scope

- Перевод клиентов на новую модель API ключей с управлением сроком действия.
- Миграция без даунтайма, с обратимой схемой изменений.

## Migration Steps

1. Подготовить совместимые изменения схемы БД (additive migration).
2. Включить двойную запись для старого и нового форматов токенов.
3. Выполнить backfill исторических данных ключей.
4. Переключить чтение на новый формат и отключить legacy-путь.

## Rollback

- При деградации по SLO (`api-key-auth-latency` или `api-key-auth-availability`) отключить feature flag `platform.api_keys.enabled`.
- Вернуть чтение на legacy-путь в конфигурации приложения.
- Остановить backfill и восстановить обработку только старой схемы.
